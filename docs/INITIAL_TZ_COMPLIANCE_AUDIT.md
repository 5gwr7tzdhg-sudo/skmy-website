Дата: 2026-06-22. Режим: read-only; код, база, CSS и маршруты не изменялись.

Первоначальное единое ТЗ отдельным файлом не найдено. Аудит основан на согласованных требованиях и текущем коде проекта.

| Пункт | Статус | Аудит |

## Особое внимание: анимации и визуальный UX

| Пункт | Статус | Аудит |
|---|---|---|
| Анимации при прокрутке | Не реализовано | Нет IntersectionObserver, reveal-классов, scroll JavaScript или активных keyframes. |
| Эффекты появления блоков | Не реализовано | Секции не получают enter-анимаций при попадании во viewport. |
| Hover-эффекты | Реализовано | Есть небольшие CSS transition для кнопок, guide-card и article-card. |
| Живой фон | Реализовано частично | Текущий hero имеет статичный многослойный gradient/radial background; движения или реакции на scroll нет. |
| Карточки путеводителя | Реализовано | Категории и статьи имеют адаптивные карточки, ссылки и hover-реакцию. |
| UX главной: быстрый вход в помощь | Реализовано | Hero, поиск, быстрые действия, категории, инструкции и новости доступны. |
| UX главной: образ организации | Реализовано частично | Название ассоциации есть, но большая часть страницы занята карточками, разделами и инструкциями. |
| Мобильная версия public | Реализовано частично | Есть breakpoints 850/700/560px, сетки становятся одноколоночными, header переносится. Реальный device-test не проводился. |
| Мобильная версия admin | Реализовано частично | Sidebar превращается в верхний блок, таблицы получают горизонтальный scroll. Реальный device-test не проводился. |
| SEO | Реализовано частично | Техническая основа готова; результат зависит от production BASE_URL, контента, Search Console и реальных переводов. |

## Почему текущая главная воспринимается как каталог

После hero подряд расположены быстрая помощь, популярные инструкции, категории путеводителя и новости. Три первых блока используют сетки карточек и ведут к материалам. Это хорошо для поиска ответа, но создаёт образ базы знаний, а не организации.

Для общественной организации на странице недостаточно представлены:

- миссия и причины существования SKMY;
- реальные люди и сообщество;
- проекты, деятельность и партнёрства;
- история, опыт и результаты работы;
- основания доверия к организации.

## Технический долг

В style.css остаются неиспользуемые стили отменённых вариантов Public UI: home-v2, org-* и home-org-v3. Текущий home.html использует home-hero и home-section, поэтому эти правила не должны менять текущий рендер, но усложняют поддержку CSS.

## Вывод

Функциональное ТЗ в основном реализовано: сайт, путеводитель, поиск, SEO, admin и безопасность работают. Не реализованы scroll-анимации и эффекты появления. Частично реализованы мобильная верификация, многоязычное наполнение, production-проверка и представление SKMY как общественной организации на главной.
