# SKMY MVP: текущий статус

## Назначение проекта

SKMY — многоязычный сайт для поддержки глухих иммигрантов в Финляндии. Его ключевая задача — дать понятные инструкции, официальные ссылки, новости и простой способ связаться с организацией.

## Что уже реализовано

- Базовая структура Flask-проекта с публичными и admin blueprints.
- PostgreSQL как основная база данных.
- SQLAlchemy-модели для пользователей, журнала действий, обращений, новостей, категорий путеводителя и статей путеводителя.
- Многоязычные маршруты для `fi`, `ru` и `en`.
- Публичные страницы: главная, о нас, новости, путеводитель, сурдоперевод и контакты.
- Путеводитель, который читает активные категории и опубликованные статьи из PostgreSQL.
- Создание, редактирование, публикация и скрытие категорий и статей через админ-панель.
- Вход администратора через Flask-Login; доступ к admin-разделам защищён ролью `admin`.
- CSRF-защита для всех POST-форм, включая login, CRUD-формы и быстрые действия публикации.
- Audit Log: фиксируются изменения категорий, статей, новостей и отметка прочитанного обращения.
- Контактная форма: публичное сохранение обращений и admin-просмотр с отметкой `read`.
- Модуль новостей: список, детальная публичная страница, создание, редактирование и публикация через admin.
- Поиск по опубликованным статьям путеводителя текущего языка через `ilike`.
- Базовое техническое SEO: `robots.txt`, `sitemap.xml`, canonical, OpenGraph, hreflang и SEO-стратегия.
- Поддержка FAQ в `GuideArticle.content` с отображением FAQ-блока и Schema.org FAQPage JSON-LD.
- Улучшенная главная страница: hero, поиск, быстрые действия, популярные инструкции, категории, последние новости и социальные ссылки-заглушки.

## Что входит в текущий MVP

Текущий MVP позволяет посетителю:

- выбрать язык и пользоваться публичными страницами;
- получать базовую информацию о SKMY;
- искать инструкции по путеводителю;
- читать опубликованные инструкции и новости;
- отправлять обращение через контактную форму.

Текущий MVP позволяет администратору:

- войти в защищённую админ-панель;
- управлять категориями и статьями путеводителя;
- управлять новостями;
- просматривать контактные обращения и отмечать их прочитанными;
- просматривать журнал действий.

## Что осталось до первого релиза

- Подготовить нормальные, проверенные реальные тексты для FI/RU/EN.
- Заменить заглушки Instagram и Telegram на реальные ссылки.
- Подготовить production-конфигурацию Flask и PostgreSQL.
- Настроить реальный домен и заменить временный локальный base URL.
- Развернуть проект на VPS.
- Настроить регулярное резервное копирование PostgreSQL и проверить восстановление.
- Подключить Matomo Analytics с учётом требований приватности.
- Проверить мобильную версию на реальных устройствах и популярных браузерах.
- Провести accessibility review: клавиатурная навигация, контраст, подписи форм, семантика и screen reader.
- Провести security review: production SECRET_KEY, HTTPS, cookie settings, права доступа к БД, ограничение debug-режима и обновление зависимостей.

## Что не добавлять сейчас

Следующие функции не входят в первый релиз и не должны расширять текущий MVP без отдельного решения:

- личные кабинеты;
- CRM;
- Telegram Bot;
- AI assistant;
- сложная система мероприятий;
- платежи;
- форум.

## Следующий технический этап

Следующая задача — подготовить проект к безопасному production-развёртыванию:

1. Добавить production-настройки и разделить development/production конфигурации.
2. Создать `.env.example` без секретов.
3. Создать `docs/DEPLOYMENT.md` с инструкцией развёртывания на VPS.
4. Создать или обновить `docs/SECURITY.md` с правилами хранения секретов, доступа и обновлений.
5. Добавить backup script для PostgreSQL и документировать процедуру восстановления.

После этого можно переходить к развёртыванию на тестовый production-сервер и финальной проверке контента.
