# Production checklist: SKMY

Перед запуском `skmy.fi` отметьте каждый пункт после фактической проверки.

## Приложение и конфигурация

- [ ] `Debug=False` в production.
- [ ] Production `SECRET_KEY` задан и отличается от development-значения.
- [ ] `.env` имеет права доступа только для пользователя приложения.
- [ ] `.env` и backup-файлы исключены из Git.
- [ ] PostgreSQL использует отдельного пользователя и пароль.
- [ ] Gunicorn/Waitress запускается как отдельный непривилегированный пользователь.
- [ ] systemd-сервис включён и запускается после перезагрузки сервера.

## Сеть и HTTPS

- [ ] DNS для `skmy.fi` настроен на VPS.
- [ ] Nginx reverse proxy настроен и проверен.
- [ ] HTTPS включён, SSL-сертификат валиден.
- [ ] HTTP перенаправляется на HTTPS.
- [ ] HTTPS-only cookies включены после проверки production-конфигурации.

## Данные и восстановление

- [ ] PostgreSQL backup настроен по расписанию.
- [ ] Backup uploads настроен или задокументирован на момент появления uploads.
- [ ] Защищённый backup `.env` создан.
- [ ] Восстановление PostgreSQL проверено на тестовой среде.

## Функциональность и доступ

- [ ] Главная, путеводитель, поиск, новости и контакты открываются.
- [ ] Login администратора работает.
- [ ] CSRF работает: POST без токена отклоняется.
- [ ] Admin закрыт для неаутентифицированных пользователей.
- [ ] Audit Log записывает admin-действия.
- [ ] Sitemap работает: `/sitemap.xml`.
- [ ] Robots работает: `/robots.txt`.

## SEO, мобильность и безопасность

- [ ] Canonical, hreflang и OpenGraph проверены на production-домене.
- [ ] Sitemap и robots используют `https://skmy.fi`, а не локальный URL.
- [ ] SEO проверен в Google Search Console после запуска.
- [ ] Мобильная версия проверена на реальных устройствах и браузерах.
- [ ] Accessibility review выполнен.
- [ ] Security review выполнен: rate limiting, headers, session settings и права доступа.
